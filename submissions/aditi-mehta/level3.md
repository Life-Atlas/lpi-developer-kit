## What I Built

**The SMILE Gap Analyzer (Critic Agent)**

Instead of a basic chatbot that just answers the user question, I built the SMILE Gap Analyzer. It acts as a Senior Architect. When a user submits a digital twin idea, this agent audits the concept against the SMILE framework to find physical, human, and architectural blind spots.

I also engineered a **stateless IPC bridge** using Python's `subprocess` to solve `BrokenPipeError` issues, and added a Regex-based security layer to sanitize terminal inputs.

---

## How to Run It

```bash
git clone [https://github.com/Dynamic-ctrl/lpi-smile-agent](https://github.com/Dynamic-ctrl/lpi-smile-agent)
cd lpi-smile-agent

# Update LPI_SERVER_PATH in agent.py to point to your local build
python3 agent.py "I want to build a digital twin of an offshore wind farm"
```

## Explainability

Every architectural critique generated by the agent contains an inline citation:
```text
[SOURCE: LPI/smile_overview] — Overview suggests potential safety hazards for pedestrians.
[SOURCE: LPI/query_knowledge] — Knowledge base indicates staff manual overrides are not addressed.
[SOURCE: LPI/get_case_studies] — Case studies reveal power outages can affect sensor accuracy.
```

At the end of the execution, the agent prints a full provenance summary:
```text
PROVENANCE — Every critique traced to its LPI source
[1] Tool: smile_overview     Finding: Baseline architectural safety hazards...
[2] Tool: query_knowledge    Finding: Specific manual override constraints...
[3] Tool: get_case_studies   Finding: Past failure metrics (power outages)...
[4] Tool: smile_phase_detail Finding: Sensor implementation gaps...
```
